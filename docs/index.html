Squatch Framework

The missing link in css frameworks. Everything you need, nothing more.

Download Squatch

 

or install with git:

$ git clone https://github.com/thesquatchman/squatch-sass.git
$ cd squatch-sass
$ npm install
$ gulp

Why use Squatch?

Haven't Bootstrap and Foundation pretty much got you covered? Yes, both these frameworks offer lots and lots of features, more than a small website needs.

Lightweight

Squatch is designed to give you more freedom and flexibility without overloading you components that need to be customized.

Quantity != Quality

Any responsive website needs certain basic components, navs, buttons, grids, etc. we focus on doing these things well.

Be original

With our flexible and intuitive sass files, we put the design decisions back on the developers and designers where they belong.

Using the Sass files:

Squatch uses sass maps to make customization much faster:

Colors

Edit the '$colors' variable and you'll get a bunch of helpful classes for text and background colors. Add or remove as many as you need and call them anything you like:

$colors: (
 grey: #2C3E50,
 red: #E74C3C,
 off-white: #ECF0F1,
 light-blue: #3498DB,
 blue: #2980B9
);

In the sass files you can use the 'color' function to get your shade:

.myClass {
 background: color(off-white);
}

 

Squatch will auto-generate text color and background classes from all the colors in you map:

This paragraph has a class of .text-blue

This span has a class of .bg-blue

This paragraph has a class of .text-red

This span has a class of .bg-red

Responsive Breakpoints

Edit the '$breakpoints' variable and squatch will re-generate all the media queries for flexbox grid, utilities like hidden classes, etc:

$breakpoint-unit: 'px';
$breakpoints: (
 sm : 480,
 md : 768,
 lg : 1024,
);
$breakpoint-navbar: 768 !default;

 

With these default values, use the class 'hidden-min' or 'hidden-max' appended by any of your breakpoint keys:

<button class="btn hidden-min-md" >
 Click to call!
</button>

This element will be set to 'display: none' on screens with a minimun width of 768px

Squatch is flexible!

Flexbox grid system makes designing responsive layouts a breeze. Float based grids are no longer needed. Autoprefixer will take care of the differences in modern browsers. IE 9 and below are dead to us. Go ahead and click the examples below to play with different layouts you can create just by adding classes to a containing element and child elements:

Layout Classes

Apply these flexbox properties to container elements by adding the class in gray: (* browser default)

  • Flex Direction
  • row* class: row
  • column class: col
  • row-reverse class: row-rev
  • column-reverse class: col-rev
  • Flex Wrap
  • nowrap* class: nowrap
  • wrap class: wrap
  • wrap-reverse class: wrap-rev
  • Gutters
  • Gutters class: gutters

Item Width:

Item width can be set by adding a class '.size1 - .size12' to any element with an '.item' class. The width is set by 'flex-basis' rather than the 'width' property, so it will behave a little differently.

  • size2
  • size4
  • size6
  • size8

Screen Sizes:

Prefix the item size with the media query breakpoint and to specify a width for the minimum screen size.

  • md2
  • md4
  • md6
  • md8

Order:

Add classes of 'first' and 'last' to change the order, or 'first-md' etc. to apply on minimum screen size.

  • first
  • last

Rethinking Responsive Layouts

Every Flexbox class has media queries for each of your set breakpoints (ie 'row', 'row-sm', 'row-md'). Let's say you want to create a two column layout with a main content area and a sidebar. The sidebar should display below the content up to a medium sized screen, then it needs to be to the left of the content. Here's what the html might look like:

<header>...</header>
  <div id="wrapper" class="col row-md">
    <article class="item md8">
    ...article content...
    </article>
    <aside class="item first-md">
    ...sidebar content...
    </aside>
  </div>
<footer>...</footer>

One Javascript Function...

...to rule them all, one function to .find() them , and in the DOM to .bind() them...(so so sorry)

Only one jQuery function comes with Squatch, but it is an extremely useful one. We assume you will import other libraries for adding fanciness to your site so Squatch stays out of the way. Here's how all the functionality on this demo page was done using only data-toggle="" and data-target="" in the html:

Example #1

Add the attributes to a button, anchor tag, span or whatever:. The value of data-toggle="" gets added or removed as a class from the element in the data-target="". The button gets a class of 'active' toggled on or off.

<button class="btn" data-toggle="open" data-target="#blue-box">
 open dialog
</button>
<div id="blue-box" class="">
  <p>hi there!</p>
</div>

hi there!

Example #2

Leave out the data-target and the parent li will be the target, useful for dropdown menus.

<nav class="navbar">
 <ul class="nav">
  <li><a href="#" data-toggle="open">DropDown</a>
   <ul class="dropdown-nav">
    <li><a>Example1</a></li>
    <li><a>Example2</a></li>
    <li><a>Example3</a></li>
   </ul>
  </li>
 </ul>
</nav>

Example #3

If you put the data-target on the parent ul, then the values from sibling child elements will be removed with the toggle.

Example #4

Put an id selector in the data-toggle and leave off the data-target, and it will toggle a class of 'active' on the corresponding element while removing the 'active' class from it's siblings. Useful for Tabs:

<nav>
 <ul class="nav inline">
  <li data-toggle="#tab1" class="active">Tab 1</li>
  <li data-toggle="#tab2" class="">Tab 2</li>
  <li data-toggle="#tab3" class="">Tab 3</li>
 </ul>
</nav>

<section class="tabs">
 <div id="tab1" class="tab active">
  <p>...</p>
 </div>
 <div id="tab2" class="tab">
  <p>...</p>
 </div>
 <div id="tab3" class="tab">
  <p>...</p>
 </div>
</section>
